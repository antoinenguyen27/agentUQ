      continue, annotate the trace, regenerate a risky span, retry the step, dry-run

        <strong>Localized risk, not one opaque score</strong>
        Catch brittle SQL clauses, tool arguments, selectors, URLs, paths, shell flags,
        and JSON leaves before execution.
      </li>
      <li>
        <strong>Probability honesty over fake certainty</strong>
        Use canonical scoring only for genuinely greedy runs and realized scoring for
        the path that actually came out.
      </li>
      <li>
        <strong>Cheap first-pass gating</strong>
        Put a lightweight reliability layer in front of slower systems such as retrieval,
        semantic verification, sandboxing, or human review.

  <section className="agentuq-home__section">
    <h2>What decisions AgentUQ can trigger</h2>
    <div className="agentuq-home__mini-grid">
      <div>
        <ul>
          <li><code>continue</code>: proceed normally.</li>
          <li><code>continue_with_annotation</code>: proceed, but attach the result to logs, traces, or monitoring.</li>
          <li><code>regenerate_segment</code>: retry only the risky leaf or clause when structured repair is available.</li>
          <li><code>retry_step</code> / <code>retry_step_with_constraints</code>: rerun the whole model step.</li>
        </ul>
      </div>
      <div>
        <ul>
          <li><code>dry_run_verify</code>: run a safe validator before execution.</li>
          <li><code>ask_user_confirmation</code>: pause before a side effect.</li>
          <li><code>block_execution</code>: fail closed before anything external happens.</li>
          <li><code>custom</code>, <code>emit_webhook</code>, and <code>escalate_to_human</code>: dispatch through your own policy layer.</li>
        </ul>
      </div>
